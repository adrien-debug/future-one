⚡ Quick Start

Installation

# Clone repository
git clone https://github.com/adrien-debug/future-one.git
cd future-one

# Frontend setup
cd frontend
cp .env.example .env.local
npm install
npm run dev  # Port 3000

# Backend setup (new terminal)
cd ../backend
cp .env.example .env
npm install
npm run dev  # Port 4000

Environment Variables

Frontend (.env.local):

NEXT_PUBLIC_SUPABASE_URL=https://zrvlmhuymhyrzonnihce.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
NEXT_PUBLIC_API_URL=https://future-one-api-production.up.railway.app/api

Backend (.env):

PORT=4000
SUPABASE_URL=https://zrvlmhuymhyrzonnihce.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...
FRONTEND_URL=https://frontend-6d793z4b4-adrien-nejkovics-projects.vercel.app

🚀 Deployment

Workflow CI/CD

  1. Git Push: git push origin main
  2. Vercel: Auto-deploy frontend (trigger on push)
  3. Railway: Auto-deploy backend (trigger on push)
  4. Verification: Check health endpoints + test contact form

Manual Deploy

# Frontend
cd frontend
npm run build
vercel --prod

# Backend
cd backend
npm run build
railway up

📝 Décisions Techniques (ADRs)

ADR-001: Next.js 16 (App Router)

Décision: Next.js 16 avec App Router (React Server Components)

Raisons: Performance RSC, SEO natif, file-based routing, intégration Vercel

ADR-002: Express.js 5 (Backend)

Décision: Express.js 5 avec TypeScript

Raisons: Simplicité, maturité, middleware ecosystem, suffisant pour charge actuelle

ADR-003: Supabase (Database)

Décision: Supabase (PostgreSQL managé)

Raisons: PostgreSQL standard, RLS natif, REST API, excellent DX, free tier généreux

ADR-004: Tailwind CSS 4

Décision: Tailwind CSS 4

Raisons: Utility-first, design tokens CSS variables, JIT, responsive mobile-first

ADR-005: TypeScript Strict

Décision: TypeScript 5.7 en mode strict

Raisons: Type safety, IntelliSense, refactoring safe, types = documentation

🔐 Security & Compliance

Implemented

  • ✅ HTTPS only — TLS 1.3 on all connections
  • ✅ CORS restricted — Only *.vercel.app + configured origins
  • ✅ Helmet middleware — Security headers (CSP, HSTS, etc.)
  • ✅ RLS (Row Level Security) — Supabase tables protected
  • ✅ Input validation — All user inputs sanitized
  • ✅ No secrets in code — .env files gitignored
